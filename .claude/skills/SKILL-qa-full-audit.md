## Блок 1: Базовый флоу (критично для запуска)

### 1.1 Генерация КП
```
Проверить вручную (http://localhost:3000):
□ Форма открывается без ошибок в консоли
□ Все 6 полей заполняются текстом
□ Кнопки тона работают (Официальный / Дружелюбный / Агрессивный)
□ Загрузка логотипа — файл принимается, превью показывается
□ Кнопка "Создать КП" → loading spinner появляется
□ Через 3-8 сек — редирект на /result
□ На /result текст КП отображается полностью
□ Нет [object Object] или undefined в тексте
□ Счётчик "Осталось X из 3" уменьшается на 1

Playwright команда:
npx playwright test tests/e2e/generate.spec.ts --headed
```

### 1.2 PDF / Печать
```
□ На /result — 4 шаблона отображаются как карточки
□ Классика — кликабельна, превью меняется
□ Минимализм — кликабельна, превью меняется
□ ВИП — показывает замок PRO если нет оплаты
□ Современный — показывает замок PRO если нет оплаты
□ Кнопка "Печать/PDF" → открывает print dialog
□ В print preview: документ выглядит как КП (не как веб-страница)
□ PDF сохраняется без обрезки текста

ВНИМАНИЕ: @react-pdf/renderer в package.json но не используется.
Печать идёт через window.print() — убедись что CSS для print настроен.
Проверить: есть ли @media print { ... } в стилях.
```

### 1.3 Редактирование и копирование
```
□ Кнопка "Редактировать" — текст становится редактируемым (textarea)
□ Изменение текста → обновляет превью
□ Кнопка "Копировать" → текст в буфере (проверить вставкой в Notepad)
□ После редактирования печать — отображает изменённый текст
```

### 1.4 История КП
```
□ После генерации КП → запись появляется на главной странице
□ /dashboard — история отображается
□ Статус КП меняется (Черновик → Отправлено → Принято)
□ Удаление КП — работает с подтверждением
□ После обновления страницы — история не теряется

ПРОВЕРИТЬ КРИТИЧНО: история в localStorage или на сервере?
Если localStorage — данные потеряются при смене браузера/устройства.
```

## Блок 3: Оплата (главная проблема)

```
ДИАГНОСТИКА ОПЛАТЫ — выполнить по шагам:

Шаг 1: Проверить env переменные
□ Открыть Vercel Dashboard → Settings → Environment Variables
□ YUKASSA_SHOP_ID — это реальный ID из личного кабинета ЮКасса?
□ YUKASSA_SECRET_KEY — это реальный ключ? (начинается с "live_" или "test_")
□ Если значения "test" или пустые → это причина ошибки

Шаг 2: Проверить fallback логику
□ Открыть lib/yookassa.ts
□ Найти условие: if (!YUKASSA_SHOP_ID || YUKASSA_SHOP_ID === 'test')
□ Если такого условия нет — fallback не срабатывает → ДОБАВИТЬ

Шаг 3: Проверить webhook
□ В личном кабинете ЮКасса есть URL webhook?
□ URL должен быть: https://[твой-домен]/api/webhooks/yookassa
□ Без webhook: деньги списываются, но план не активируется

Шаг 4: Тест mock оплаты (без реальных денег)
□ Временно выставить YUKASSA_SHOP_ID='' в .env.local
□ PaywallModal → нажать купить → должен сработать mock
□ После mock оплаты: план должен активироваться

Playwright команда:
npx playwright test tests/e2e/payment.spec.ts --headed

Ожидаемый результат:
✅ Mock оплата → план активируется → PRO шаблоны доступны
```

## Блок 4: Лимиты и тарифы

```
□ Бесплатно: 3 КП, после — paywall
□ Paywall показывается при попытке создать 4-е КП
□ Paywall показывается при выборе ВИП/Современный шаблона
□ После оплаты Старт (199₽): доступно 10 КП + ВИП
□ После оплаты Активный (490₽): доступно 30 КП + все шаблоны
□ После Безлимит (790₽/мес): нет ограничений
□ Счётчик в хедере обновляется после оплаты

⚠️ БЕЗОПАСНОСТЬ: Кредиты в localStorage — пользователь может
открыть DevTools → Application → localStorage → изменить kp_credits.
Это обходит всю монетизацию. Серверная проверка критична.

Проверить: /api/user/credits синхронизирует план?
□ Открыть Network tab → обновить /result → есть ли запрос к /api/user/credits?
□ Ответ содержит актуальный план из Vercel KV?
```

## Блок 6: Скорость и ошибки

```
Performance:
□ Lighthouse score > 70 (открыть DevTools → Lighthouse → Analyze)
□ Время до первого байта < 3 сек (проверить в Network tab)
□ DeepSeek генерация: spinner показывается сразу
□ Если DeepSeek завис > 10 сек: есть таймаут и friendly сообщение

Ошибки:
□ Открыть Console (F12) → НЕТ красных ошибок при загрузке
□ НЕТ 404 на ресурсы (шрифты, иконки, изображения)
□ При ошибке DeepSeek: показывается "попробуйте ещё раз", не пустой экран
□ При ошибке оплаты: конкретное сообщение, не "undefined"
```
