## Следующий спринт — Бэклог

### [PDF] Серверная генерация PDF — полные поля на всех страницах

**Проблема:**
Chrome при печати через `window.print()`:
- При `@page { margin: 0 }` — нет URL/даты в PDF, но страница 2+ начинается без отступа сверху
- При `@page { margin: 14mm }` — правильные отступы, но Chrome добавляет URL и номера страниц

**Решение (следующий спринт):**
Реализовать серверную генерацию PDF через Puppeteer или `@react-pdf/renderer`:

| Вариант | Сложность | Качество |
|---|---|---|
| `@react-pdf/renderer` (уже в package.json) | Средняя | Высокое — полный контроль |
| Puppeteer на Vercel Functions | Высокая | Высокое — точный рендер |
| Playwright PDF | Высокая | Высокое |

**Рекомендуемый путь:** `@react-pdf/renderer` — уже установлен, не требует headless браузера.

**Что нужно сделать:**
1. Создать PDF-версию каждого шаблона через `@react-pdf/renderer` (компоненты `Document`, `Page`, `View`, `Text`)
2. Реализовать `app/api/pdf/route.ts` — принимает kpId, возвращает PDF файл
3. Кнопка "Скачать PDF" делает GET `/api/pdf?id=xxx` вместо `window.print()`
4. Убрать print CSS из globals.css (больше не нужен)

**Примерный объём:** 1 день работы
